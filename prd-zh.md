# Birder Studio — 产品需求文档

## 愿景

Birder Studio 是一款为观鸟者打造的 AI 驱动的拍摄后工作流工具。它将观鸟中最痛苦的环节——从数百张照片中筛选整理——变成一个快速、愉悦的体验。

**一句话概括**：导入照片，获得一个经过筛选、物种标记、可直接分享的图库。

## 为什么要做这个产品

每次观鸟结束后，观鸟者都面临同样繁琐的流程：

1. 从相机导入 200-500+ 张照片
2. 手动翻阅找出好片（大多数是模糊的、重复的、或者只拍到空树枝）
3. 尝试辨认不清晰照片中的鸟种
4. 编辑精选照片
5. 撰写图文说明，发布到社交媒体或观鸟社区（eBird、Merlin、论坛）

这个过程要花好几个小时。步骤 1-3 完全是苦力活。现有工具没有一个真正"懂鸟"——Lightroom 能整理照片但分不清柳莺和麻雀，Merlin 能通过声音识别鸟种但无法批量处理单反照片。

Birder Studio 就是相机和分享之间缺失的那个工具。

## 目标用户

**核心用户**：拥有专业摄影器材（单反/微单 + 长焦镜头）的资深观鸟爱好者。他们每周出去拍鸟 2-4 次，每次拍摄数百张照片，并积极在 eBird、Instagram、观鸟论坛或个人博客上分享。

**为什么不是休闲观鸟者？** 用手机拍鸟的人照片少、需求简单——Apple Photos 就够用了。我们的价值随照片量和物种识别需求而增长。

**为什么不是职业野生动物摄影师？** 他们已经有成熟的 Lightroom/Capture One 工作流，不会切换。我们是补充他们的工作流，而非取代。

## 核心产品原则

1. **以观鸟行程为核心，而非单张照片** — 最小单元是一次观鸟行程（日期 + 地点 + 一批照片），而非单张照片。一切从"我刚观鸟回来"开始。
2. **AI 做无聊的活** — 筛选、去重、物种识别、质量评分全部自动完成。观鸟者只做创意决策，不做分类决策。
3. **有主见的默认值，轻松覆盖** — AI 挑选最佳照片并识别物种。观鸟者随时可以修正，但大多数时候不需要。
4. **快速分享** — 从导入到发布社交媒体应该只需几分钟，而非几小时。

## MVP 功能集

### 核心循环：导入行程 → AI 筛选 → 审阅 → 分享

#### 1. 行程导入
- 拖放文件夹（或从 Finder 选择）
- 自动提取 EXIF：日期、时间、GPS 位置、相机参数
- 自动创建行程实体（日期 + 通过反向地理编码获取的地点名称）
- 支持 RAW 格式（CR2、CR3、NEF、ARW、RAF）+ JPEG/HEIF

#### 2. AI 筛选（杀手级功能）
- **重复/连拍分组** — 感知哈希算法聚类近似照片
- **质量评分** — 从清晰度、曝光、构图、鸟类可见度等维度评分
- **自动淘汰** — 标记明显的废片（完全模糊、空画面、过曝），观鸟者一键清除
- **组内最佳选择** — 从 15 张类似连拍中，高亮最好的 1-2 张
- **物种识别** — 使用 Vision/Core ML 自动标记鸟种。显示置信度。支持"不确定"标记

观鸟者的审阅体验：打开行程，看到 AI 高亮的精选，快速确认/拒绝，完成。

#### 3. 行程图库
- 按物种组织的网格视图
- 星级评分（1-5）标记个人收藏
- 并排对比模式，用于在相似照片间选择
- EXIF 元数据面板（焦距、快门速度、ISO — 观鸟者很在意这些）
- 物种信息面板（中文名、英文名、学名、保护状态）

#### 4. 快速编辑
- 基础调整：曝光、对比度、饱和度、白平衡、锐度
- 裁剪（三分法和黄金比例辅助线）
- 一键水印（自定义文字/Logo、位置、透明度）
- **不是**完整的照片编辑器 — 我们补充 Lightroom，而非替代它

#### 5. AI 内容生成
- 选择一张照片 → 生成社交媒体文案：
  - **Twitter/X**：简短有趣的文案，包含物种名称、地点、鸟类趣知识
  - **Instagram**：较长的图文说明，带话题标签和故事性
  - **eBird 记录备注**：客观的观察记录，包含行为、数量、栖息地
  - **博客/文章**：关于该物种的科普文章
- 观鸟者编辑 AI 草稿，复制到剪贴板，粘贴到对应平台
- 由 Claude API 驱动，使用观鸟领域专属提示词

### MVP 不包含的功能

- 视频编辑（Phase 2）
- 直接发布到社交媒体 — v1 用剪贴板就够了
- 衍生品制作（印刷品、周边）
- 观鸟者之间的协作/分享
- iOS/iPad 版本
- 云同步
- eBird API 集成自动提交清单（Phase 2，大机会）

## 平台与分发

- **macOS 14+（Sonoma）** — 原生 SwiftUI 应用
- **为什么目前只做 macOS**：我们的目标用户在桌面处理照片。macOS 提供最佳的图像处理性能（Metal、Core Image）、原生文件系统访问，以及通过 Apple 框架实现的 RAW 格式支持。
- **分发方式**：先从 birding.me 直接下载，之后上架 Mac App Store（沙盒限制可能影响文件访问）

## 商业模式（初步构想）

**免费增值 + 行程次数限制：**
- **免费版**：每月 5 次行程，基础 AI 筛选，无内容生成
- **Pro 版**（$9.99/月 或 $79.99/年）：无限行程，完整 AI 筛选，内容生成，水印
- **终身版**：$199 一次性买断（早鸟价）

逻辑：订阅费用覆盖内容生成的 Claude API 成本。AI 筛选使用本地模型（无逐用户成本），因此免费版可持续运营。

## 架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 平台 | 原生 SwiftUI | 图像密集工作流的性能保障，macOS API 访问 |
| 持久化 | SwiftData | 现代、Swift 原生、满足本地优先应用需求 |
| 图像处理 | Core Image + Metal | GPU 加速，支持 RAW，macOS 自带 |
| 鸟种识别 | Core ML（端侧） | 快速、离线可用、无逐张 API 成本 |
| 内容生成 | Claude API | 最佳写作质量，理解语境和语气 |
| AI 筛选/评分 | 端侧（Vision + Core ML） | 必须快且离线可用 — 不能把 500 张照片发到 API |

## 核心指标（如何衡量产品是否有效）

- **行程完成率** — 导入行程后完成审阅并保留精选照片的比例
- **从导入到首次分享的时间** — 目标：200 张照片的行程在 5 分钟内完成
- **AI 准确率** — 物种识别准确率、筛选误伤率（好照片被标记为差）
- **留存率** — 每周活跃行程数/用户

## birding.me 域名定位

- **birding.me** = 营销网站 + 账户管理 + 许可证激活
- 应用本身本地优先，核心功能无需云端依赖
- 未来：可选的精选图库云备份，通过 birding.me/username 分享公开图库

## 竞品分析

| 工具 | 功能 | 我们填补的空白 |
|------|------|----------------|
| Adobe Lightroom | 专业照片编辑 + 管理 | 不懂鸟、无 AI 筛选、无物种识别 |
| Apple Photos | 消费级照片管理 | 无观鸟功能、RAW 支持弱、无 AI 筛选 |
| Merlin Bird ID | 通过声音/照片识别鸟种 | 一次只能识别一张、无批量处理、无编辑、无内容生成 |
| eBird | 观鸟清单 + 社区 | 无照片管理，纯文字清单 |
| Canva | 社交媒体内容制作 | 通用工具、无观鸟语境、无照片筛选 |

**我们的独特定位**：唯一一款专为观鸟者拍摄后照片工作流打造的工具，内置真正理解鸟类的 AI。
