# Kestrel — 产品计划书 v0.1

> 筛选、精修、创作——一个应用，全部搞定。

## 愿景

Kestrel 是面向观鸟者的全能 Mac 桌面应用。它用一个极速原生应用取代 4-5 个割裂的工具，将筛选、精修、创作、分享整合在一起。不强制顺序，不限定流程——你需要做什么，它就在那里。

目标：比鸟友现有的任何工具好 10 倍。不是靠功能堆砌，而是三件事同时做到：每个功能本身就极其好用，功能之间无缝衔接，并且新手和专业人士都能各取所需。就像 Figma、Mac、iPhone——简单的事情一看就会，复杂的事情做得到且做得优雅。

## 目标用户

**核心用户：** 活跃的观鸟者，每周至少拍摄一次，单次产出数千张照片，并在社交媒体或观鸟社群分享。他们有不错的器材，但不是专业修图师——他们想要好效果，但不想学 Lightroom。

**次要用户：** 鸟会组织者、自然教育者、公民科学贡献者，需要高效处理和呈现鸟类影像。

## 设计原则

1. **每个功能都极其好用** — 不是"能用就行"，而是每个功能都打磨到让人惊喜。裁切不只是裁切，是懂鸟的裁切。导出不只是导出，是一键出大片。
2. **一个应用，所有事情** — 筛选、精修、创作都在同一个地方完成。不强制顺序，不限定流程。你可以从筛选开始，也可以直接拿一张老照片做明信片。像 Notion 之于文档工作，我们之于鸟类影像工作。
3. **新手秒上手，专家不嫌浅** — 像 Figma 和 iPhone：打开就会用，但深挖下去功能极其强大。简单操作一键完成，复杂操作做得到且做得优雅。
4. **鸟为核心的智能** — 每个功能都知道画面里有一只鸟。自动裁切、自动标注、自动增强都围绕主体展开。
5. **速度即功能** — 每个交互亚秒级响应。没有转圈，没有等待。
6. **美是标配** — 每个输出默认就好看。模板、预设、导出都经过精心设计。

## 产品参考：五个标杆，五条法则

我们不重新发明轮子。以下五个产品在各自领域做到了极致，我们从每个身上学一件最重要的事。

### Pixelmator Pro — "ML 是隐形的魔法，不是展示的功能"

Pixelmator Pro 最聪明的地方不是它用了 ML，而是你感觉不到它用了 ML。你点"自动白平衡"，它就对了。你点"超分辨率"，图就清晰了。没有参数，没有进度条，没有"AI 正在处理"的提示。ML 是手段，不是卖点。

很多产品犯的错是把 AI 当 feature 来秀——"看，我用了 AI！"。Pixelmator 从来不这样。它的态度是：你不需要知道背后发生了什么，结果好就行。

**我们怎么用：** 我们的鸟类检测、质量评分、物种识别，用户不需要知道背后跑了什么模型。他们只需要看到：照片自动按质量排好了，物种名字自动标上了，裁切自动对准鸟了。如果用户需要理解 ML 才能用好我们的产品，那就是我们的失败。

### Darkroom — "同一界面服务新手和专家"

Darkroom 做对的一件事是同一个界面服务两种人。新手看到的是几个滑块——亮度、对比度、饱和度，左右滑就行。专家看到的是色彩曲线、HSL、选择性调整。这两层不是在设置里切换"简单模式/专业模式"，而是自然嵌套的。基础调整就在那里，点开"更多"才看到高级的。

另一个亮点：批量编辑。你调好一张图的参数，可以一键应用到整组。这对我们极其重要——鸟友一次拍几百张同一场景的照片，光线和环境都一样，不应该每张都调一遍。

**我们怎么用：** 精修阶段的 UI 不要做两个模式，要做一个渐进式界面。默认展示 5 个最常用的调整（裁切、曝光、白平衡、锐化、饱和度），往下或往里点才出现降噪、曲线、选择性编辑。场景内的批量应用必须是一等公民——调好一张，同场景全部跟上。

### Linear — "快到让你忘记它是个软件"

Linear 的核心洞察是：软件的速度直接影响用户的思维速度。当一个工具足够快，你就不再"操作工具"，而是直接在"思考问题"。

Linear 做了几件事：所有操作即时响应，没有 loading 状态；键盘快捷键覆盖所有操作，手不用离开键盘；`Cmd+K` 万能搜索，任何东西都能找到；动画极短（100-150ms），给你反馈但不浪费时间。还有一个很少人提的：Linear 的默认值极好。创建一个 issue，不需要填 12 个字段。标题，回车，完了。

**我们怎么用：** 筛选阶段要做到 Linear 的速度感。鸟友筛 2000 张照片，如果每张要等 0.5 秒才能看下一张，那就是 1000 秒的等待——不可接受。必须做到：按键即切换，预加载下一张，手势和键盘都丝滑。`Cmd+K` 全局搜索也应该有——搜物种、搜日期、搜地点、搜项目，一个入口搞定。

### Figma — "功能是被发现的，不是被教的"

Figma 最厉害的不是功能多，而是学习曲线的设计。第一次打开 Figma，你能画个矩形、写个字。但你用了一个月后发现 Auto Layout，用了三个月发现 Components，用了半年发现 Variables。每一层都是在你需要的时候自然出现的，而不是一个 30 分钟的新手教程强塞给你的。

背后的设计哲学是：界面的信息密度要和用户的熟练度成正比。新手看到简洁的界面，因为高级功能藏在深处。老手看到丰富的界面，因为他们知道去哪里找。

另一个洞察：Figma 几乎没有弹窗。所有操作都在画布上原地完成。右键菜单、属性面板、图层列表——都是画布的延伸，不是独立的对话框。

**我们怎么用：** 两件事。第一，不要做新手教程。如果需要教程才能用，就是设计失败。功能应该在上下文中自然浮现——比如用户选中一张鸟照，旁边就出现"智能裁切"的选项，而不是让用户去菜单里找。第二，避免弹窗。导出设置、水印设置、标注编辑，都应该在主界面里原地完成，不要弹出新窗口打断沉浸感。

### Loom — "把十步变成一步"

Loom 之前，录屏幕分享给同事需要：打开 QuickTime -> 选择录制区域 -> 录制 -> 停止 -> 保存文件 -> 压缩 -> 上传 -> 复制链接 -> 发送。十步。Loom 之后：点一下开始，点一下结束，链接自动在剪贴板里。两步。

它的洞察是：大多数用户不需要中间步骤的控制权。他们不需要选编码格式，不需要选分辨率，不需要手动上传。他们只要结果。

**我们怎么用：** 这直接指导我们的"一键鸟像模式"和"社交就绪"导出。现在鸟友发一张照片到社交媒体需要：打开编辑器 -> 裁切 -> 调曝光 -> 调白平衡 -> 锐化 -> 加水印 -> 导出 -> 选格式 -> 选质量 -> 上传。我们应该做到：选中照片 -> 点"分享" -> 完成。中间的裁切、增强、水印、格式转换全部自动处理，用智能默认值。当然，每一步都可以手动调，但默认路径就是一键。

### 速查表

| 从谁学 | 核心法则 | 落地到我们的产品 |
|---|---|---|
| Pixelmator Pro | ML 是隐形的魔法，不是展示的功能 | 鸟类检测、评分、识别都无感发生 |
| Darkroom | 同一界面服务新手和专家，批量编辑 | 渐进式编辑 UI + 场景级批量应用 |
| Linear | 速度即思维，默认值要好 | 筛选必须即时响应，所有操作都有好的默认值 |
| Figma | 功能被发现而非被教，不用弹窗 | 无教程，上下文浮现功能，主界面内完成一切 |
| Loom | 把十步变一步 | "一键出图"：选照片，点分享，完成 |

## 技术架构

### 技术栈

| 层级 | 技术方案 | 选择理由 |
|---|---|---|
| UI 界面 | SwiftUI | 原生 Mac 体验，流畅动画，内建无障碍支持 |
| 图像处理 | Core Image + Metal | GPU 加速滤镜，实时预览 |
| ML / 鸟类检测 | Core ML | 模型体积小，设备端推理，硬件加速 |
| 数据存储 | SwiftData / SQLite | 快速本地数据库，存储照片元数据、标签、项目 |
| 文件处理 | Uniform Type Identifiers | 通过 ImageIO 原生支持 RAW 格式（CR2, CR3, NEF, ARW, DNG） |
| 导出 | PDFKit + Core Graphics | 印刷级输出，用于周边模板 |
| AI 生成 | Stable Diffusion (Core ML) / 外部 API | 风格转换，插画生成 |

### 模型迁移

现有的 PyTorch 模型（用于鸟类检测的 Mask R-CNN、质量分类器）将通过 `coremltools` 转换为 Core ML 格式。这能显著缩小模型体积，并在 Apple Silicon 上启用 Neural Engine 加速。

## 三大阶段

## 阶段一：筛选 — "从混沌到清晰"

智能分拣系统。解决最痛苦的第一步：把 2,000 张原片变成 50 张精选。

### 核心功能

| 功能 | 描述 | 优先级 |
|---|---|---|
| 自动导入 | 监听文件夹或 SD 卡挂载，自动导入 | P0 |
| 鸟类检测 | 使用 Core ML 定位画面中的鸟 | P0 |
| 质量评分 | 评估锐度、噪点、运动模糊、曝光 | P0 |
| 场景分组 | 按时间戳 + 视觉相似度聚类连拍序列 | P0 |
| 物种识别 | 自动分类物种，显示中文名 + 学名 | P0 |
| 快速筛选 UI | 滑动/键盘驱动的接受/拒绝，组内最佳优先展示 | P0 |
| 多主体 | 处理多鸟画面，独立评分每只鸟 | P1 |
| RAW 支持 | CR2, CR3, NEF, ARW, DNG, RAF 原生解码 | P0 |
| 智能过滤 | 按物种、质量等级、日期、机身、镜头过滤 | P1 |
| 筛选统计 | "你从 2,341 张中保留了 47 张，检测到 12 个物种" | P2 |

### 用户流程

```
SD 卡 / 文件夹
    |
    v
[导入] --> [检测鸟类] --> [质量评分] --> [场景分组]
    |
    v
[筛选视图：逐场景审阅，最佳优先，滑动接受/拒绝]
    |
    v
"已接受"照片池 --> 流入阶段二
```

## 阶段二：精修 — "从留片到佳作"

专为鸟类摄影打造的轻量编辑器。不是 Lightroom——更快、更简单、更懂鸟。

### 核心功能

| 功能 | 描述 | 优先级 |
|---|---|---|
| 智能裁切 | 自动检测鸟的位置，按三分法建议最优裁切 | P0 |
| 曝光 / 亮度 | 基础曝光校正，保护高光不过曝 | P0 |
| 白平衡 | 预设（日光、阴天、阴影、金色时刻）+ 手动调节 | P0 |
| 锐化 | 羽毛细节感知锐化（锐化主体，不锐化背景） | P0 |
| 降噪 | 基于 ML 的降噪，保留羽毛细节 | P1 |
| 饱和度 / 自然饱和度 | 细腻的羽色增强 | P1 |
| 背景虚化 | 加深背景散景，隔离主体 | P1 |
| 眼部提亮 | 自动检测并为鸟眼添加眼神光 | P1 |
| 主体感知暗角 | 压暗边缘但保持鸟的正常曝光 | P1 |
| 背景清理 | 去除/弱化杂乱的枝条、电线 | P2 |
| 栖枝清理 | 轻量仿制/修复鸟周围的小干扰物 | P2 |

### 一键预设

杀手级功能：理解鸟是主体的智能预设。

| 预设 | 效果 |
|---|---|
| 鸟像模式 | 自动裁切 + 羽毛锐化 + 背景虚化 + 眼部提亮 + 暗角 |
| 图鉴模式 | 紧凑裁切，中性白平衡，平光，全身可见 |
| 戏剧模式 | 高对比，深阴影，暖高光，强暗角 |
| 清新明亮 | 提亮阴影，冷色调，最少处理，编辑感 |
| 社交就绪 | 平台适配裁切 + 细微增强 + 水印 |

## UI / UX 方向

### 布局理念

单窗口应用，三条"车道"（筛选 / 精修 / 创作）通过顶部导航切换。照片从左到右流过管线。当前阶段始终可见，其他阶段一键即达。

```
+-----------------------------------------------------------------------+
|  [筛选]          [精修]          [创作]            [照片库]              |
+-----------------------------------------------------------------------+
|                                                                       |
|  侧边栏：          主画布：              检查器：                       |
|  - 场景列表        - 照片全尺寸视图      - 元数据                       |
|  - 物种分类        - 编辑控件            - 快捷操作                     |
|  - 过滤器          - 模板预览            - 导出选项                     |
|                                                                       |
+-----------------------------------------------------------------------+
|  胶片条 / 缩略图栏                                                     |
+-----------------------------------------------------------------------+
```

### 视觉风格

- **色彩：** 低饱和大地色系作为 UI 底色，鲜艳的鸟照作为主角。UI 应当退后，鸟应当跳出。
- **字体：** SF Pro（系统字体）用于 UI，可选衬线体用于标签/叠加层（博物学家质感）。
- **图标：** 细线条图标，适当融入鸟类元素但绝不俗气。
- **动效：** 细腻的物理动画。照片滑动、扇开、堆叠。没有生硬的过渡。
- **默认深色模式：** 鸟友在野外拍了一天后晚上修图。默认深色，提供浅色选项。
- **信息密度：** 默认舒适，为小屏幕高级用户提供紧凑模式。
